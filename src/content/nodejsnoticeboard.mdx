##  주요 기능

<div className="space-y-6 mt-4">  
  <div className="space-y-2">
    ![메인 페이지](/nodeimg/nodemain.png)
    <div className="text-base text-gray-800">
      게시글 목록은 GET / 요청을 통해 fs.readdir로 ./data 디렉토리의 파일 목록을 불러오고, 
      이를 HTML 목록으로 변환해 클라이언트에 렌더링합니다.메인 페이지(GET /)에 접속하면 서버는 fs.readdir을 통해 ./data 디렉토리 내 모든 파일명을 가져옵니다. 
      각 파일명은 게시글의 제목으로 간주되며, HTML 
      &lt;ul&gt; 태그 형태로 동적으로 렌더링됩니다. 사용자는 목록에서 제목을 클릭해 해당 게시글의 상세 내용을 확인할 수 있습니다.
    </div>
  </div>

  <div className="space-y-2">
    ![글 작성 페이지](/nodeimg/nodecreact.png)
    <div className="text-base text-gray-800">
       사용자가 '글 작성하기' 버튼을 클릭하면 /create 경로의 글 작성 폼으로 이동합니다. 제목과 내용을 입력 후 제출하면, POST /create_process 요청으로 서버에 데이터가 전송됩니다. 
       서버는 fs.writeFile을 이용해 해당 데이터를 ./data 폴더에 새 파일로 저장합니다. 저장된 파일의 이름은 제목이고, 파일의 내용은 작성한 본문이 됩니다
    </div>
  </div>

  <div className="space-y-2">
    ![글 수정 페이지](/nodeimg/Editpost.png)
    <div className="text-base text-gray-800">
     사용자가 게시글 상세 페이지에서 '수정' 버튼을 클릭하면 /update?id=제목 경로로 이동합니다. 수정 폼에는 기존 제목과 내용이 미리 입력되어 있어 쉽게 수정할 수 있습니다. 수정 후 제출하면, 
     서버는 POST /update_process 요청을 받아 fs.rename으로 파일명을 변경하고, fs.writeFile로 내용을 덮어씌웁니다. 제목까지 수정 가능하도록 구현되어 있습니다.
    </div>
  </div>

  <div className="space-y-2">
    ![글 삭제 페이지](/nodeimg/dlet.png)
    <div className="text-base text-gray-800">
      상세 페이지에서 '삭제' 버튼을 클릭하면, POST /delete_process 요청이 서버로 전송됩니다. 삭제 버튼을 누르기 전에는 자바스크립트의 window.confirm 함수로 사용자에게 한 번 더 확인을 받습니다. 
      서버는 fs.unlink를 사용하여 해당 파일(게시글)을 삭제하고, 삭제 후에는 메인 페이지(/)로 자동 이동됩니다.
    </div>
  </div>

</div>
