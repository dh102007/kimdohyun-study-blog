## 주요기능

<div className="space-y-6 mt-4">

  <div className="space-y-2">
    ![메인 페이지](/img/main.png)
    <div className="text-base  text-gray-800">
      게시글 목록을 확인할 때는 GET /api/get 요청으로 모든 게시글 데이터를 불러오며, 서버는 이를 JSON 형태로 응답합니다. 클라이언트에서는 해당 데이터를 받아 화면에 렌더링합니다.
    </div>
  </div>

  <div className="space-y-2">
    ![글 작성 페이지](/img/min.png)
    <div className="text-base  text-gray-800">
      사용자가 클라이언트에서 게시글을 작성하면, 제목, 내용, 작성 날짜를 포함한 데이터를 POST /api/insert 엔드포인트로 서버에 전송합니다. 
      서버는 이 데이터를 INSERT 쿼리를 통해 MySQL 데이터베이스의 kimawsdb 테이블에 저장합니다.
    </div>
  </div>

  <div className="space-y-2">
    ![글 수정 페이지](/img/sujun.png)
    <div className="text-base text-gray-800">
      특정 게시글을 수정하고자 할 때는 글 목록에서 해당 항목을 클릭하여 수정 페이지로 이동한 후, 수정된 제목과 내용, 
      idx(식별자)를 PUT /api/update 엔드포인트로 전송합니다. 서버는 이를 기반으로 데이터베이스에서 업데이트합니다.
    </div>
  </div>

  <div className="space-y-2">
    ![글 삭제 페이지](/img/dlet.png)
    <div className="text-base  text-gray-800">
      게시글 삭제 기능은 DELETE /api/delete/:idx 엔드포인트를 통해 구현되었으며, 게시글 식별자 idx를 URL 파라미터로 받아 해당 게시글을 삭제합니다. 
      삭제 요청이 완료되면 클라이언트는 자동으로 목록을 새로고침합니다.
    </div>
  </div>
</div>
